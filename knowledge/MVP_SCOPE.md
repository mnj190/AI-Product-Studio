# MVP Scope

이 문서는 첫 번째 웹 버전에서 만들 것과 만들지 않을 것을 구분한다.

## MVP Goal

Markdown으로 작성한 개인 지식 문서를 웹에서 탐색할 수 있는 최소 포트폴리오를 만든다.

질문형 인터페이스는 중요한 기능이지만, 첫 번째 MVP에서는 LLM Wiki 문서 구조와 페이지 탐색을 먼저 완성한다.

## Must Have

첫 번째 웹 버전에 반드시 포함한다.

- 홈 페이지
- About 요약
- 프로젝트 목록
- 프로젝트 상세
- Build Log 목록
- Build Log 상세
- Markdown 기반 콘텐츠 로딩
- 기본 내비게이션
- 모바일에서도 읽기 쉬운 레이아웃

## Should Have

가능하면 첫 번째 버전에 포함한다.

- Prompt Library 목록
- Prompt 상세
- AI Stack 페이지
- Wiki 목록
- 문서 검색 전 단계의 간단한 태그 또는 카테고리

## Later

첫 번째 웹 버전 이후에 만든다.

- LLM Wiki 기반 Ask About Me 인터페이스
- 검색 인프라 또는 Vector DB 연동
- 로그인
- 관리자 페이지
- 실시간 자동매매봇 대시보드
- WebGPU 게임 데모
- MCP/Agent 실험 자동화

## Out of Scope for MVP

초기 버전에서는 하지 않는다.

- 민감한 개인정보 저장
- API Key 또는 토큰 저장
- 투자 추천처럼 보이는 자동매매 콘텐츠
- 과도한 애니메이션
- 완벽한 디자인 시스템
- 다국어 지원

## MVP Success Criteria

다음이 가능하면 MVP 성공으로 본다.

- 방문자가 조정민이 어떤 개발자인지 이해할 수 있다.
- 방문자가 주요 프로젝트를 읽을 수 있다.
- 방문자가 AI 활용 방향을 이해할 수 있다.
- 새 Markdown 문서를 추가하기 쉬운 구조이다.
- 이후 LLM Wiki와 질문형 인터페이스로 그대로 확장할 수 있다.

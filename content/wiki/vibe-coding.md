# Vibe Coding

## Summary

Vibe Coding은 AI와 대화하면서 빠르게 아이디어를 구현하고 개선하는 개발 방식이다.

이 프로젝트에서는 Vibe Coding을 단순히 “AI에게 코드를 시키는 방식”이 아니라, 기획, 설계, 구현, 검증, 문서화까지 이어지는 협업 방식으로 본다.

## How I Use It

현재는 다음 흐름을 선호한다.

1. ChatGPT로 방향과 정보 구조를 잡는다.
2. Codex나 Claude Code로 실제 구현을 진행한다.
3. 구현 중 생긴 결정과 문제를 Build Log에 남긴다.
4. 좋은 프롬프트는 Prompt Library에 저장한다.
5. 결과물을 다시 문서화하고 챗봇 지식으로 연결한다.

## What Matters

Vibe Coding에서 중요한 것은 빠르게 만드는 것만이 아니다.

중요한 기준:

- 요구사항을 제대로 정리했는가?
- AI가 참고할 컨텍스트를 충분히 제공했는가?
- 결과물을 검증했는가?
- 실패와 의사결정을 기록했는가?
- 다음 작업자가 이어받기 쉬운가?

## Risks

AI를 빠르게 쓰다 보면 다음 문제가 생길 수 있다.

- 방향 없이 코드만 늘어난다.
- 문서와 실제 구현이 어긋난다.
- AI가 추측한 내용을 사실처럼 받아들인다.
- 보안 정보가 실수로 저장된다.
- 결과 검증 없이 넘어간다.

## My Rule

AI는 빠르게 시도하게 해주는 도구이지만, 최종 책임은 개발자에게 있다.

그래서 이 프로젝트에서는 작업 규칙, 문서 구조, 보안 기준, Build Log를 함께 관리한다.

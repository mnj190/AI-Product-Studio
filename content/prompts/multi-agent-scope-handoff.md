# Multi-Agent Scope Handoff Prompt

## Purpose

여러 AI 에이전트가 같은 포트폴리오나 워크스페이스를 나눠 작업할 때, 담당 범위를 분리하고 남은 작업을 안전하게 이어가기 위한 프롬프트이다.

## Used In

- Trading Bot 작업을 별도 에이전트 담당 범위로 분리
- 기본 포트폴리오, LLM Wiki, Project Archive 작업 계속 진행
- Safe Local Backlog에서 외부 결정 없이 진행 가능한 작업 선별

## AI Tool

- Codex
- Claude Code
- ChatGPT

## Prompt

```text
[프로젝트/영역 A]는 다른 에이전트가 담당하니까 제외하고,
그 외 작업의 현재 진행 상황과 앞으로 해야 할 일을 정리해줘.

이후 현재 워크스페이스 상태를 기준으로 외부 결정이나 민감 정보가 필요 없는 작업부터 하나씩 진행해줘.

진행할 때는 다음 기준을 지켜줘.

1. 제외한 영역의 코드와 문서는 새로 확장하지 않는다.
2. 기존 변경 사항을 되돌리지 않는다.
3. 남은 작업은 작은 단위로 나눈다.
4. 구현, 문서화, 검증 결과를 함께 남긴다.
5. 다음 에이전트가 이어받을 수 있게 작업 로그와 Next Actions를 갱신한다.
```

## When To Use

- 같은 저장소에서 여러 에이전트가 병렬로 작업할 때
- 한 프로젝트는 다른 사람이 맡고, 나머지 영역만 이어서 진행해야 할 때
- 작업 범위가 섞여 충돌이 날 가능성이 있을 때
- “계속 진행” 요청 전에 제외할 영역을 명확히 하고 싶을 때

## Why It Works

이 프롬프트는 AI에게 “무엇을 할지”뿐 아니라 “무엇을 하지 않을지”를 함께 알려준다.

멀티 에이전트 작업에서는 빠른 진행보다 범위 분리가 더 중요할 때가 있다. 담당 영역을 먼저 나누면 서로의 변경을 덮어쓰거나, 민감한 프로젝트의 공개 기준을 다른 흐름에서 accidentally 바꾸는 일을 줄일 수 있다.

또한 현재 워크스페이스를 기준으로 다시 확인하게 만들기 때문에 이전 대화 기억보다 실제 파일 상태에 근거해 이어갈 수 있다.

## Reusable Pattern

```text
[제외할 영역]은 [담당자/에이전트]가 맡을 예정이야.

그 영역은 건드리지 말고, 나머지 작업만 현재 레포 상태 기준으로 정리한 뒤 계속 진행해줘.

먼저:
- 현재 git 상태
- Next Actions
- 최근 Build Log
- 관련 정책 문서

를 확인하고, 외부 권한이나 사용자 결정이 필요 없는 작은 작업부터 처리해줘.

각 작업은 코드/문서 변경, 검증, 다음 액션 갱신까지 포함해서 마무리해줘.
```

## Notes

담당 범위 분리는 단순한 일정 관리가 아니라 품질 관리다.

특히 공개 포트폴리오와 비공개 프로젝트가 같은 워크스페이스에서 연결될 때는, 어떤 에이전트가 어떤 기준을 바꾸는지 명확히 남겨야 한다.

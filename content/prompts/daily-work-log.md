# Daily Work Log Prompt

AI와 함께 한 작업을 매일 아침 공유용 기록으로 정리하기 위한 프롬프트다.

## Prompt

```text
어제 한 작업 또는 아직 기록되지 않은 모든 작업을 확인해서 공유용 Daily Digest로 정리해줘.

목적은 AI로 어떤 식으로, 어떻게, 얼마나 작업했는지 보여주는 거야.

먼저 아래를 확인해줘.

1. git 상태와 최근 커밋
2. 어제 날짜의 Build Log와 Daily Digest 존재 여부
3. 기록되지 않은 변경 사항이나 의사결정
4. 공개 로그에 남길 만한 중요한 요청/프롬프트
5. 빌드, 화면, API 등 검증 결과

그 다음 아래 내용을 포함해서 작성해줘.

1. 어제 만든 결과물 요약
2. AI를 사용한 방식: 기획, 구현, 검증, 기록으로 구분
3. 사람이 직접 결정한 중요한 방향 전환
4. 바이브 코딩 초보자가 참고할 만한 요청/프롬프트
5. 그 프롬프트가 왜 효과 있었는지
6. 검증한 내용
7. 아직 남은 일과 다음에 이어갈 가장 작은 작업

기존 Build Log가 있으면 하루 단위로 묶고, 빠진 작업이 있으면 먼저 작업 로그를 추가해줘.
민감한 정보나 비공개 정보는 공개 로그에 넣지 말고 요약해줘.
```

## When To Use

- 매일 아침 작업 기록을 남길 때
- 여러 개의 작은 작업 로그를 하루 요약으로 묶고 싶을 때
- AI 협업 과정을 포트폴리오 콘텐츠로 바꾸고 싶을 때
- 바이브 코딩을 처음 하는 사람에게 과정을 보여주고 싶을 때

## Why It Works

이 프롬프트는 단순히 “무엇을 했는지”가 아니라 “AI를 어떻게 사용했는지”를 기록하게 만든다.

결과물, 의사결정, 프롬프트, 검증을 함께 남기기 때문에 나중에 포트폴리오, 블로그, 회고, 강의 자료로 재사용하기 좋다.

특히 `git status`, 최근 커밋, 기존 Build Log를 먼저 확인하게 만들기 때문에 “기억나는 것만 쓰는 회고”가 아니라 실제 변경 사항에 근거한 기록이 된다.

## Reusable Pattern

```text
[기간] 동안 한 작업 또는 아직 기록되지 않은 작업을 확인해서 [공유 목적]에 맞는 기록으로 정리해줘.

먼저 확인할 것:
- git 상태와 최근 커밋
- 기존 작업 로그와 하루 요약
- 빠진 의사결정이나 검증 결과
- 공개할 만한 주요 프롬프트

포함할 내용:
- 만든 결과
- AI를 사용한 방식: 기획/구현/검증/기록
- 사람이 결정한 것
- 중요한 요청/프롬프트
- 의사결정
- 검증
- 다음 작업

민감한 정보는 제외하고, 처음 보는 사람도 흐름을 이해할 수 있게 써줘.
```

## Short Morning Version

매일 아침에는 아래처럼 짧게 요청해도 된다.

```text
어제 작업 기록해줘. 기록되지 않은 작업까지 확인해서 공유용 Daily Digest로 정리하고, 중요한 프롬프트는 초보자도 따라 할 수 있게 남겨줘.
```
